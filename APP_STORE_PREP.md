# خطة تجهيز the-sfm trader للنشر لاحقاً في App Store

قبل النشر العام نحتاج تحويل النسخة من تجربة خاصة إلى منتج جاهز للمراجعة.

## المطلوب

1. استضافة backend على HTTPS ثابت.
2. تغيير `server.url` في `capacitor.config.json` إلى رابط HTTPS.
3. تشغيل:
   ```bash
   npm run ios:sync
   ```
4. إضافة Privacy Policy.
5. تجهيز Screenshots للأحجام المطلوبة في App Store Connect.
6. كتابة وصف واضح:
   - التطبيق يعرض تحليلات تعليمية.
   - ليس مستشاراً مالياً.
   - المستخدم مسؤول عن قرارات التداول.
7. اختبار الصوت والتنبيهات على جهاز فعلي.
8. رفع TestFlight أولاً.
9. بعد استقرار النسخة، إرسالها للمراجعة.

## تنبيه مراجعة Apple

التطبيقات المالية تحتاج وضوح في:

- مصدر البيانات.
- حدود الدقة.
- المخاطر.
- عدم تقديم وعود أرباح.
- سياسة الخصوصية.

## نص إخلاء مسؤولية مقترح

```text
the-sfm trader يقدم تحليلات سوق آلية لأغراض تعليمية ومتابعة شخصية فقط. لا يعتبر التطبيق نصيحة مالية أو توصية استثمارية ملزمة. قرارات البيع والشراء مسؤولية المستخدم وحده، وقد تكون بيانات السوق متأخرة أو غير مكتملة.
```
